🧠 Prompt Start:

You are now building the Gravity Logic Engine for OrbitDeck.

⸻

🎯 Goal

Enable OrbitDeck to intelligently suggest what content a user should view next — based on what they’ve clicked, what tags exist, and what Claude infers from content relationships.

This system powers:
	•	SuggestedNext panel
	•	Orbit reordering logic
	•	Content relevance mapping

⸻

🧩 Content Structure (Required)

Each slide/node uses this format:

type Slide = {
  title: string
  content: string
  tags: string[]
  media?: string
  gravity?: {
    related?: string[]    // Titles of related nodes
    intensity?: number    // Optional influence score
  }
}

If gravity.related exists, use it.
If not, fallback to tag-matching to determine next nodes.

⸻

🔄 Gravity Logic Flow

When a user clicks on a node:
	1.	That node becomes the focused center
	2.	The tags of that node are evaluated
	3.	All other nodes are scored based on:
	•	Shared tags (simple count)
	•	Optional Claude-inferred gravity.related
	4.	The 3 best matches are:
	•	Moved closer in the orbit (optional animation)
	•	Shown in the <SuggestedNext /> panel

⸻

⚙️ Core Function

Build a function like:

function getSuggestedNext(current: Slide, all: Slide[]): Slide[] {
  if (current.gravity?.related?.length) {
    return all.filter(slide =>
      current.gravity.related?.includes(slide.title)
    ).slice(0, 3)
  }

  return all
    .filter(slide => slide.title !== current.title)
    .map(slide => ({
      ...slide,
      relevance: slide.tags.filter(tag =>
        current.tags.includes(tag)
      ).length
    }))
    .sort((a, b) => b.relevance - a.relevance)
    .slice(0, 3)
}
